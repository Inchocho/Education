프로젝트 진행 순서도

구현해야할 기능

1. 
일반유저
	회원 가입 (회원가입 폼 -> 화면단에서 정규표현식) // 최우선 필요 
		ㄴ 회원가입의 경우 필수폼만 입력하면 가입이 됨		
	로그인
		ㄴ 회원가입 로그인 후 추가입력폼에 값이 없을경우 추가입력폼(선택) - 장르,성별,자기소개등 입력폼이 생성됨 	
	비밀번호 찾기
		ㄴ 회원가입 후 이메일을 입력하면 해당 이메일이 DB에 있는지를 확인하고 이메일로 임시비밀번호를 전송함

2.
관리자
	영화등록(단독 가능)
		ㄴ 영화정보를 등록받고 사진 업로드 필요	
	영화관리(영화등록이 우선시됨)
		ㄴ 영화정보를 수정,삭제
	회원관리
		ㄴ 회원에 대한 정보를 수정(필수적으로 회원이 필요하므로 회원가입이 필요함)
	환불관리
		ㄴ 회원이 신청한 환불에 대해 환불처리를 하므로 회원가입 + 회원의 환불요청이 필요함 	
	게시글 관리	(신고받은 게시글 삭제처리)
		ㄴ 회원이 작성한 게시글에 신고를 하면 게시글관리에 쌓이는 방식으로 
		  회원가입 및 회원의 게시글 작성, 그리고 게시글 신고가 필요함 
		  
3.
공통기능
	검색옵션, 검색
	(게시글의 경우 검색옵션: 작성자, 제목, 전체)
	(영화의 경우 검색옵션: 작성자, 영화제목, 장르...)

공통헤더
	바로가기기능
	순위/평점/장르/추천작품(메인화면 목록을 보여주는 화면인데 저게 검색옵션임) 게시판 
	검색, 내정보, 로그아웃

ㄴ 관리자헤더 	
	바로가기기능
	영화등록/영화관리/회원관리/환불관리/게시글관리
	
4.
내정보(마이페이지)	(회원 캐쉬 // 회원 상세)
	회원 캐쉬 
		ㄴ 회원가입이 우선시됨 
	회원 상세
		ㄴ 회원가입이 우선시됨 , 회원정보 수정과 비밀번호 수정이 가능함 

5.
영화 정보(목록, 상세, 상세2)
	필수적으로 관리자 영화등록이 완성되야함 
	영화 목록, 상세, 상세2 자체가 관리자 영화등록을 통해 DB에 등록된 영화를 보여주는 방식임
	(많은 영화 데이터가 들어와있어야 함)
	
	영화진흥회 API 연결되면 여기서 빼던가 
	아니면 직접 입력해서 일단 목록을 만들어야함

6.
회원 관리
	
	

정리

특징
1. 기본적으로 사이트자체가 로그인을 해야만 작업이 가능함 

관리자는 우선적으로 DB에서 계정을 생성해서 회원가입 로직이 따로 없음 
유저는 최초 
로그인폼 -> 회원가입 -> 사이트 이용

즉 모든 기능이 로그인을 해서 사용이 가능 (관리자 파트 일부 제외)

관리자
영화등록, 영화관리(수정,삭제)의 경우 단독으로 작업이 가능함 

나머지 환불관리, 게시물신고관리는 유저로부터 게시글이나 구매내역에 대해 환불요청이 올경우 처리하므로 불가

회원 - 최우선으로 회원가입, 로그인이 필요 

관리자 - 영화관리, 영화등록 특히 영화등록이 최우선적으로 작업되야함 (대부분 기능이 등록된 영화조회에 기반함)
