Log4J

자바 기반 로깅 유틸리티이다. 아파티 소프트웨어 재단의 프로젝트 
아파치 로깅 서비스(Apache Logging Services)의 일부이다.
Log4j는 여러 자바 로깅 프레임워크들 가운데 하나이다.
소프트웨어 개발자가 프로그램을 개발하는 도중에 디버깅등을 위해 
정해진 양식에 맞추어 화면 상이나 파일로 로그를 남길 목적으로 사용된다

현재는 SLF4J와 Logback이 많이 사용되고 있다

*Log level (로그 출력등급)
Trace	:	Debug 보다 더 상세한 정보
Debug	:	애플리케이션의 내부 실행 상황을 추적해 볼 수 있는 상세 정보 ##최초 개발시 사용됨
Info	:	애플리케이션의 주요 실행 정보	##중요한 정보만 보여줌 ex)파라미터나 데이터단위
Warn	:	잠재적인 위험을 안고 있는 상태	
Error	:	오류가 발생했지만, 애플리케이션은 계속 실행할 수 있는 상태 
Fatal	:	애플리케이션을 중지해야 할 심각한 오류

보통 Debug,Info,Error를 자주사용하게 될것
상위 로깅메소드사용시 하위 로깅메소드의 로그도 전부 출력된다(즉 Trace에 1개 Debug에 2개 Warn에 3개 있을시 Trace를 사용하면 6개가 나옴)

프로그램에 sysout은 사용하지 않고 log4J를 통해 확인한다
